## 今天最值得关注的 3 个产品

### 1. Firecrawl Research Index - 16/18

- Product Hunt：<https://www.producthunt.com/products/extract-by-firecrawl>
- 官网/官方来源：<https://firecrawl.dev/blog/research-index-launch>
- 核实信息：官方介绍称它是面向 agentic AI/ML research 的专用索引，覆盖论文检索、全文片段验证、相关论文扩展、GitHub artifacts 检索，并通过 API、CLI、MCP、SDK 接入。
- 为什么值得关注：它把“研究型 agent”的关键瓶颈从普通网页搜索推进到可验证的论文与代码检索闭环，适合观察 deep research agent 如何从检索走向证据核验和实现参考。
- 评分：相关度 5，机制新颖度 5，设计启发 5，市场信号 1。

### 2. API to MCP - 15/18

- Product Hunt：<https://www.producthunt.com/products/api-to-mcp>
- 官网：<https://apitomcp.ai/>
- 核实信息：官网显示它可把 REST/GraphQL、SaaS 与内部业务 API 转成托管远程 HTTP MCP server，支持 OAuth、API key、Bearer、Basic Auth、workflow tools、JMESPath 输出映射，并提供 Visual Builder 与 Agent Builder 两条创建路径。
- 为什么值得关注：MCP 的落地难点之一是把真实 API 安全、可控、可测试地暴露给 agent。API to MCP 直接切这个基础设施层，对“让既有软件变成 agent tool surface”有较强参考价值。
- 评分：相关度 5，机制新颖度 4，设计启发 5，市场信号 1。

### 3. VELA - 15/18

- Product Hunt：<https://www.producthunt.com/products/vela-7>
- 官网：<https://vela-secure.vercel.app>
- 核实信息：Product Hunt 描述为“Securely execute AI-generated & untrusted code”；官网显示其核心是面向 AI agents 与 SaaS 的安全执行 runtime，使用 Firecracker micro-VM、HMAC capability tokens、policy engine、intrusion detection、audit trail，并提供 LangChain、LlamaIndex、CrewAI、OpenAI 等适配。
- 为什么值得关注：随着 coding agent 和自动化 agent 需要运行模型生成代码，“在哪里执行不可信代码”会成为基础设施问题。VELA 的 micro-VM + capability token + 审计模型，对 agent sandbox 设计很有启发。
- 评分：相关度 5，机制新颖度 4，设计启发 5，市场信号 1。
