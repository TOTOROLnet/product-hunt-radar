## 初筛方法

本轮不是只做关键词匹配，而是同时判断：

1. 产品是否实质落在 AI Agent、大模型 API/开发者平台、Context Engineering/Memory/MCP/Tool Use、新型人机交互、AI 原生生产力、Coding Agent/Developer Tools。
2. 官网或官方文档是否能说明真实机制，而不是只有泛化营销页。
3. 产品是否对 agent 工具调用、上下文工程、长期记忆、协作入口、安全治理、低延迟模型或 AI 工作流产品化有启发。
4. 是否存在降权因素，例如普通 ChatGPT 套壳、图片/头像生成、轻量效率工具、Crypto/NFT、只有落地页或与关注方向弱相关。

评分规则：相关度 0-5，产品机制新颖度 0-5，对 AI 产品设计的启发 0-5，市场信号 0-3，总分 18；只保留总分 >= 11 的产品。

### 1. WorkClaw - 17/18

- Product Hunt：<https://www.producthunt.com/products/workclaw>
- 官网：<https://www.workclaw.com>
- PH 核实信号：页面标题为 “Collaborative, proactive AI coworkers who work in Slack”；PH 页面显示当日榜单第 1，`latestScore: 284`，`commentsCount: 65`，`followersCount: 457`。
- 官网核实信息：官网称其是 “the AI team for your team”，提供可协作、主动、可定制的 AI coworkers，可在 Slack 中工作，支持创建多个 WorkClaws，并强调团队协作、安全、管理控制和 OpenClaw。
- 为什么值得关注：它不是单人 Chatbot，而是把 agent 组织成可命名、可分工、可共享的“团队成员”，并放进 Slack/Teams 这类团队协作入口。对 AI 产品经理最有启发的是：agent 的单位可能从“一个助手”变成“一个可管理的虚拟组织”，产品重点也从 prompt UI 转向权限、职责、例行工作、团队可见性和安全控制。
- 如果失败，最可能输在哪里：如果实际执行能力不足、需要大量人工配置，或企业安全/权限边界无法让团队放心授权，容易退化成 Slack 里的多 bot 套壳；同时，多 agent 协作如果没有清晰责任归属，可能带来噪音和信任成本。
- 评分：相关度 5，机制新颖度 4，设计启发 5，市场信号 3。

### 3. pumaDB - 16/18

- Product Hunt：<https://www.producthunt.com/products/pumadb>
- 官网：<https://pumadb.ai>
- PH 核实信号：页面标题为 “a small hosted memory layer for AI agents”；PH 页面显示当日榜单第 5，`latestScore: 135`，`commentsCount: 6`，`followersCount: 121`。
- 官网核实信息：官网称其是 “Agent memory API without database work”，为 agents 提供 durable memory；支持 hosted MCP 或 REST，存储 JSON rows，并可查看 agent 记住的内容。官网示例明确提到 ChatGPT、Claude、Codex 与 Streamable HTTP MCP endpoint。
- 为什么值得关注：agent memory 是 context engineering 的核心基础设施之一。pumaDB 的价值不在“又一个数据库”，而在把 memory 做成 agent 可直接连接、用户/开发者可审查、无需自建 RAG/vector stack 的轻量层。
- 如果失败，最可能输在哪里：如果 memory schema、检索质量和权限隔离没有比开发者自己用数据库/vector store 更省心，产品会被视为薄封装；同时，记忆内容的编辑、遗忘、审计和多 agent 共享边界会直接决定可信度。
- 评分：相关度 5，机制新颖度 4，设计启发 5，市场信号 2。

## 趋势/信号

1. **团队协作入口正在成为 agent runtime**：WorkClaw 和 Slackbot MCP Client 都把 agent 放进 Slack/Teams，而不是让用户去新的独立 app 里工作。
2. **MCP 从开发者工具扩散到企业协作层**：Slackbot MCP Client 显示 MCP 的下一步不是只连接 IDE，而是连接组织里已有的应用、权限与多人上下文。
3. **agent memory 正在产品化为独立基础设施**：pumaDB 代表“可审查、可连接、低运维”的 memory layer，而不是每个 agent 团队都自建数据库/RAG。
4. **低延迟模型供给仍是 agent 体验瓶颈**：Mellum 的信号在于，开发者工具公司也在围绕低延迟、高性能模型布局，以支撑更实时的 AI workflow。

## 作为 AI 产品经理的跟进判断

我最想跟进 **Slackbot MCP Client + agent memory/permissions** 这个方向。理由是：MCP 让工具连接标准化，Slack/Teams 让 agent 进入真实团队流程，而 memory/权限/审计决定它是否能从 demo 进入生产。如果投入产品探索，我会优先做“团队频道里的可审计工具调用 + 共享/私有记忆 + 管理员审批”的闭环，而不是单独再做一个通用 AI 助手。
